Thread of Execution and the Call Stack

Javascript is not a asynchronous language,
it is a synchronous language with asynchronous
capabilities.
JavaScript is a single-threaded language.
Thread is a single sequential flow of
control - a process of a program:
  Thread of execution:
  op1 -> op2 -> op3 -> op4 -> Call Stack
                           -> Memory Heap

The Call Stack keeps track of functions (a stack
of functions) to be executed & manages execution
contexts. The Stacks are LIFO (last in first out).

  Call Stack :  third()
                second()
                first()
                Global EC (execution context)

A Stack is a data structure.

Example of a simple call stack:
                  function first() {
                    console.log('first...');
                  };
                  function second() {
                    console.log('second...');
                  };
                  function third() {
                    console.log('third...');
                  };
                  first();
                  second();
                  third();

                  output: first...
                          second...
                          third...

Example of a just a little more complex call stack:
                  function first() {
                    console.log('first...');
                    second();
                  };
                  function second() {
                    console.log('second...');
                    third();
                  };
                  function third() {
                    console.log('third');
                  };
                  first();

                  output: first...
                          second...
                          third...
#: call stack, function execution, data structure, JavaScript under the hood

startsWith()

To determine if a substring starts at the beginning or at a specified index of a string we can utilize the startsWith() ES6 feature (for supporting browsers). The method belongs to all string objects and primitives. It accepts two arguments: the first is a string to search for and the second is an aptional argument that determines the start position to search from which defaults to 0.

if('my string'.startsWith('m')) { console.log('The given string starts with the letter "m".'); } else { console.log('The given string does not start with the letter "m".'); } if('my string'.startsWith('s', 3)) { console.log('"my string" contains the letter "s" at index 3.'); } else { console.log('"my string" does not contain the letter "s" at index 3.'); } var name = 'Jim'; if(name.startsWith('John', 4)) { console.log('The string ' + name + ' starts with the word "John" from the 5th character.'); }

Array into String Conversion

The .toString() method creates a string separated by commas. It may be good solution to assign this doing to save the outcome (as shown with the 'var myNbrStr' below).
The .join('separator') method creates a string of the Array elements separated by a value specified by the developer ('separator').

ES6: Methods is ES6 Classes

Since ES6 it is not required to use the keyword function for a function assigned to a name of a method. The prototype method is one of the types of class methods available to objects of the class.
The code below includes a getter - area() - and a method - calculateArea().

class Triangle {
constructor(height, base) {
this.height = height;
this.base = base;
}
get area() {
return this.calculateArea();
}
calculateArea() {
return this.height * this.base * 0.5;
}
}

const triangleOne = new Triangle(5, 5);
console.log(triangleOne.area);

//Output: 12.5

Node.js - overview

Tech:
  ■ It is not a JS framework, but a JS Runtime
  ■ Just as a Google Chrome, Node.js was built on the V8 JS engine
  ■ Written in C++
  ■ Allows us to run JS code on the server

Pros:
  ► Works on a single thread using non-blocking I/O calls
  ► Asynchronous
  ► Supports thousands concurrent connections
  ► Optimizes throughout & scalability in apps with many I/O operations
  ► Fast & efficient apps

NPM - Node Package Manager:
  ▬ Installs third party packages
  ▬ Packages are stored in the "node_modules" folder
  ▬ Dependencies get listed in a "package.json" file
  ▬ NPM scripts can be written to run specific tasks such as run a server
  ▬ some commands:
	npm init 	- generates a package.json file
	npm install packageName  - installs a package locally
	npm install -g packageName  - installs a package globally

Node Modules:
  ° Node Core Modules: path, fs, http, etc
	const path = require('path');
  ° Third party modules
  ° Custom modules (files with export)
	const myFile = require('./myFile');

All the modules in Node.js are wrapped in
what is called a Module Wrapper Function:
(function (exports, require, module, __filename, __dirname){

}),
which gives acces to all the five included features, so
for example:
console.log(__dirname, __filename);
will print in the console paths to the directory of the file and the path to
the file respectively
